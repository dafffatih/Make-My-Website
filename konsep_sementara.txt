disini saya ingin menjelaskan konsep untuk bagian right side dari page chat:

1. saat user pertama kali mengakses page chat, user akan disuruh memilih projek yang dia pesan, atau kalau dia belum ada projek yang dipesan di layar hanya memunculkan pilihan new project, untuk membuat dan memesan projek web baru
2. setelah user memilih projeknya atau membuat projek barunya, user akan diarahkan ke laman dimana ada progress bar diatas, yang menginformasikan status projeknya sudah sampai mana, dan ketika user pertama kali membuat projeknya maka statusnya masih tahap pertama yaitu project setup, yang mana isinya user bisa memilih paket yang sesuai dengan tujuan websitenya dan mengisi form kebutuhan kebutuhan webnya
3. selanjutnya setelah user memilih paket dan mengisi form projeknya dengan benar maka statusnya masuk ke tahap selanjutnya atau tahap ke-2 yatiu deposit payment, user akan diminta membayar deposit projeknya minimal sebesar 20% sampai maksimal 100% biaya pembuatan websitenya, jika user membayar 100% websitnya maka user akan melewati tahap ke-4 atau menyelesaikan pembayaran secara keseluruhan
4. tahap ke-3 (in Development) user akan menunggu developer menyelesaikan websitnya, user akan diminta untuk memantau secara berkala laman ini untuk merespon developer ketika ada pertanyaan atau pilihan yang ditawarkan, dan nanti setelah pembuatannya selesai developer akan menampilkan hasilnya di laman ini, tetapi user belum bisa mengakses websitnya secara penuh karena belum di deploy
5. tahap ke-4 User akan diminta melunaskan pembayaran projeknya ini dan user juga bisa merequest domain yang diinginkan (dengan biaya terpisah), setelah pembayaran tuntas user akan langsung mendeploy websitnya dan mengsetting domainnya yang dipesan, barulah user bisa mengakses websitnya secara full
6. tahap ke-5 tahap revisi, dimana diawal user memilih paket yang tersedia terdapat fasilitas revisi yang jumlah revisinya sesuai dengan yang tertera pada paket yang dipilih, user bisa merevisi apapun pada websitenya (mulai dari merubah tulisan, mengganti desain, mengganti logo, dan sebagainya). Jadi user bisa menambahkan revisi sampai maksimal 20 revisi, dan ketika di submit maka tiket revisi akan berkurang 1, dan bisa melakukan submit revisi sampai jumlah jatah revisinya habis. Jika kuota revisinya habis user bisa melakukan pembelian tiket revisi tambahan.
7. setelah semuanya selesai user bisa memesan website baru lagi dengan klik new projek, atau bahkan ketika 1 projek masih dalam proses pembuatan user tetap bisa memesan projek baru lainnya dengan klik new projek, dengan begitu user bisa memiliki lebih dari 1 projek yang masih dalam proses.
